variables for pre-processing

polish knows several variables and switches which can be set directly or indirectly in the polish.xml file. All switches and variables start with "polish.", so you should not encounter any difficulties with your own switches or variables.

This documented is targeted at developers who wants to use polish-specific variables or switches in their own widgets or canvasses.

Switches

Switches are used to indicate specific properties. They are often mutually exclusive with others.

  • polish.images.backgroundLoad: this switch is set when images of the widgets, backgrounds and canvasses should be loaded in a background thread. This can speed up the responsiveness of an application dramatically, but it also means that some images are set while the user is in a screen. This switch is mutually exclusive with the switch polish.images.directLoad. This switch is set indirectly in polish.xml by the imageLoadStrategy-tag.
  • polish.images.directLoad: this switch is set when images of the widgets, backgrounds and canvasses should be loaded in the foreground thread. This can slow down the responsiveness of an application, but it also means that all images are loaded when the user enters a screen. This switch is mutually exclusive with the switch polish.images.backgroundLoad. This switch is set indirectly in polish.xml by the imageLoadStrategy-tag.
