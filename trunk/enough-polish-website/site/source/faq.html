Technical

Q: How much memory does J2ME Polish need?

A: J2ME Polish itself is a build-tool with an optional GUI and some helper classes. If you just use the build-tool, no additional memory is used. If you want to use the GUI the additional needed memory depends on what GUI features you actually use. The minimum library footprint starts around 15 kb (after obfuscation). For a full fledged application calculate up to 40 kb for the GUI. Modern devices have a maximum jar size of several MBs, so the memory used by the GUI is more or less negligable.

Bugs

Q: I think I have found a bug, what should I do?

A: Please tell me about it at j2mepolish@enough.de. You might also want to file a bug-report. J2ME Polish is currently in the "early bird" phase, so there are still some limitations.

Contribute

Q: I like this project, how can I contribute?

A: There are many ways how you can contribute:

  • Finding bugs
  • Translating the documentation
  • Enhancing or extending J2ME Polish
  • Creating a design with J2ME Polish

When you'd like to do anything of it, please get in touch with me at j2mepolish@enough.de.

Q: I fixed some stuff - what now?

A: Gee, cheers! Please send it to me at j2mepolish@enough.de. Be aware that I use J2ME Polish commercially as well, though.

Mailing-Lists

Q: Where are the mailing lists?

A: Here:

  • users: please join the polish-users@lists.berlios.de-list.
    In this list users can discuss their problems. The developers of the polish framework will monitor the discussions and help out!
  • announcements: please join the polish-announce@lists.berlios.de-list for retrieving news regarding the polish framework.
    This list anounces new versions and has very low traffic.
  • developers: please join the polish-devel@lists.berlios.de-list.
    This is for the developers of the polish framework.
